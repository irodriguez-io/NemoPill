  - **`PriorityOneInternetPermissionAllowListRule`** (THR-016 priority — wired first): asserts the merged manifest does not declare `android.permission.INTERNET`. Uses Konsist's manifest-parsing API or a fallback file-scan if the API does not support manifest assertions; either implementation must fail the build if `INTERNET` appears.
  - **`NoNetworkImportsRule`** (ADR-021 enforcement): asserts no class in any module imports `java.net.*`, `okhttp3.*`, `retrofit2.*`, `okhttp3.OkHttpClient`, `java.net.HttpURLConnection`, or any analogous network API. Allow-list is empty for MVP.
  - **`DomainLayerNoAndroidRule`** (ADR-009 layer direction): asserts no class under `:medication-management::domain`, `:scheduling::domain`, `:notifications::domain`, `:adherence-tracking::domain`, or `:core` (which is pure-Kotlin per ADR-009) imports `android.*`. Excludes `androidx.annotation.*` because the JSR-305 successor annotations are pure-JVM bytecode-level metadata, but explicitly does **not** allow `android.os.*` / `android.content.*` / `android.app.*`.
  - **`NoUpwardLayerDependencyRule`** (ADR-009 layer direction): asserts no `:infrastructure` package imports any `:presentation` package, and no `:application` package imports any `:presentation` package, and no `:domain` package imports any `:application` / `:infrastructure` / `:presentation` package. Konsist's package-dependency-graph API enforces this.
  - **`NoCrossFeatureDomainImportRule`** (ADR-009 feature isolation): asserts no class under `:medication-management::domain` / `:application` imports any class under `:scheduling` / `:notifications` / `:adherence-tracking`'s `:domain` / `:application`, and symmetric assertions for the other three feature modules. Cross-feature communication happens only via the `:core::DomainEventPublisher` per ADR-011 (not exercised in T-006 because no business code exists yet, but the rule is wired now so future violations fail at CI time).
  - **`PendingIntentFlagImmutableRule`** (ADR-023 enforcement): asserts every construction-site call to `PendingIntent.getBroadcast` / `getActivity` / `getService` / `getForegroundService` passes a `flags` argument whose bit-OR includes `PendingIntent.FLAG_IMMUTABLE`.
